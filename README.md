## Самостоятельное задание №1. Описание игры «Крестики-нолики»

### 1. Описание игры

**Правила игры:** Два игрока по очереди ставят X и O в пустые клетки поля 3×3. Побеждает тот, кто первым выстроит три своих символа в ряд (горизонталь, вертикаль или диагональ). Если все клетки заполнены, а победителя нет — ничья.

**Логика победы/поражения:** При победе игра останавливается и выводится сообщение о победителе. При ничьей выводится сообщение «Ничья!». После окончания игры клетки становятся неактивными.

**Особенности реализации:** Игра написана на чистом JavaScript без библиотек. Состояние хранится в массиве `board`. Интерфейс обновляется динамически через DOM.

### 2. Игровая механика

**Как определяется победитель?** Функция `checkResult()` перебирает массив `winConditions` (8 выигрышных комбинаций) и проверяет, заполнены ли три клетки одним символом.

**Как обрабатываются клики?** При клике на клетку срабатывает `handleCellClick()`. Функция проверяет, пуста ли клетка и активна ли игра. Если да — ставит символ, обновляет `board`, проверяет результат и меняет игрока.

**Как работает перезапуск?** Функция `restartGame()` сбрасывает `currentPlayer` на X, очищает `board`, делает игру активной, обновляет статус и очищает все клетки.

---

### 3. Структура данных

**Массив board:** Одномерный массив из 9 элементов. Индексы от 0 до 8 соответствуют клеткам поля. Каждый элемент хранит X, O или пробел (пустая клетка).

**Условия победы:** Массив `winConditions` содержит 8 комбинаций индексов (3 горизонтали, 3 вертикали, 2 диагонали). Победа наступает, когда все три индекса из комбинации содержат одинаковый символ.

**Состояние игры:** Хранится в переменных:
- `currentPlayer` — текущий игрок (X или O)
- `board` — состояние игрового поля
- `gameActive` — флаг активности игры (true/false)


### 4. Алгоритмы

**Алгоритм проверки победы:** 
1. Перебрать все 8 комбинаций из `winConditions`
2. Для каждой комбинации взять значения из `board` по трём индексам
3. Если все три значения одинаковые и не равны пробелу — победа

**Алгоритм определения ничьи:** 
- После проверки победы проверить, есть ли в `board` хотя бы один пробел
- Если пробелов нет и победы не было — ничья

**Алгоритм смены игрока:** 
- Если игра не закончена: `currentPlayer = currentPlayer === "X" ? "O" : "X"`
- Обновить текст статуса на `Ход игрока: {currentPlayer}`
